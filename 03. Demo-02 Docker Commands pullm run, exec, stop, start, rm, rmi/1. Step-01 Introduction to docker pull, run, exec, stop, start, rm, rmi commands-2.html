🎯 Learning Objectives ▼

By the end of this demo, you should be able to:

1. Pull a specific version of a Docker image from Docker Hub onto your local desktop.

2. Run a container from a pulled image.

3. Connect into a running container using docker exec.

4. Stop and start an existing container without recreating it.

5. Remove a container and remove an image from your local desktop.

⌨️ Command Reference ▼
1 docker pull — download an image
Pulls a tagged Docker image from Docker Hub (or your configured registry) onto your local desktop.
bash
docker pull my-nginx:v2
2 docker run — create & start a container from an image
Creates a new container from the pulled image and starts it. -d runs it in the background, --name gives it a friendly name, -p maps a host port to a container port.
bash
docker run -d --name web1 -p 8080:80 my-nginx:v2
3 docker exec — connect into a running container
Opens an interactive shell inside the running container so you can inspect it from the inside. -i keeps STDIN open, -t allocates a terminal.
bash
docker exec -it web1 bash
4 docker stop / docker start — stop & restart a container
stop gracefully stops a running container without deleting it; start brings that same container back up.
bash
docker stop web1
docker start web1
5 docker rm — remove a container
Deletes a stopped container. Add -f to force-remove a container that's still running.
bash
docker stop web1
docker rm web1
6 docker rmi — remove an image
Removes a Docker image from your local desktop. All containers created from that image must be removed first.
bash
docker rmi my-nginx:v2
CommandPurpose
docker pullDownload a Docker image from a registry (e.g. Docker Hub) to the local desktop.
docker runCreate and start a container from a local image.
docker execRun a command (often a shell) inside an already-running container.
docker stopGracefully stop a running container without deleting it.
docker startStart a container that was previously stopped.
docker rmRemove a (stopped) container from the local desktop.
docker rmiRemove a Docker image from the local desktop.
🧪 Guided Lab Exercise ▼
ℹ️
Follow Along
Run each command below in order on your own machine. Substitute my-nginx:v2 with an image you have access to (e.g. the public nginx image) if you don't have a private one.
Task 1 — Pull the image
1Run docker pull and paste the command you used, and the final line of output.
Task 2 — Run a container
2Run docker run to create and start a container. What container ID or name did Docker assign?
Task 3 — Exec into the container
3Use docker exec to open a shell inside the container. What command did you run once inside?
Task 4 — Stop and start the container
4Stop the container, verify with docker ps that it's no longer running, then start it again.
Task 5 — Remove the container and image
5Stop the container, remove it with docker rm, then remove the image with docker rmi. What error would you get if you tried docker rmi before removing the container?
📋 Summary & Coming Up Next ▼
✅
Summary — The Container Lifecycle

pull gets the image → run creates and starts a container from it → exec lets you work inside it while it runs.

stop / start pause and resume a container without losing it → rm deletes the container → rmi deletes the underlying image once no containers depend on it.
