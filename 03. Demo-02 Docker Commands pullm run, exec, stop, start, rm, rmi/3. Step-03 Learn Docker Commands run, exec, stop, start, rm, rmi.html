🎯 Learning Objectives ▼
💡
By the end of this demo, you should be able to:

1. Break down a docker run command: container name, port mapping, detached mode, image & tag.

2. List running and stopped containers with docker ps, -a, and -q.

3. Connect into a running container using docker exec, both interactively and for one-off commands.

4. Stop, start, and forcibly remove containers using name or container ID.

5. Remove an image from the local desktop with docker rmi.

▶️ docker run — Anatomy of the Command ▼
bash
docker run --name myapp1 -p 8080:80 -d stacksimplify/my-nginx:v1
1 --name myapp1 (optional)
Gives the container a friendly name instead of a random one Docker would otherwise generate.
2 -p 8080:80 — host port : container port
8080 is the host port — the port on your local desktop you'll browse to. 80 is the container port — the port the application is actually listening on inside the container.
3 -d — detached mode
Runs the container in the background instead of attaching your terminal to it, freeing your terminal to run other commands. -d can appear anywhere after docker run — it's a flag for run, not an argument tied to the image name or tag.
4 stacksimplify/my-nginx:v1 — image & tag
The image (and tag) to create the container from. If you never ran docker pull first, docker run will pull it automatically before starting the container.
ℹ️
Alternative — Run Directly From GHCR (No Pull Needed First)
bash
docker run --name myapp1 -p 8080:80 -d ghcr.io/stacksimplify/my-nginx:v1
You don't have to run docker pull first — docker run downloads the image automatically if it isn't present locally. Running docker pull as a separate first step is still worth learning, since it's a command you'll use on its own often.

📋 Listing Containers — docker ps ▼
CommandWhat It Shows
docker psOnly currently running containers.
docker ps -aAll containers, running and stopped.
docker ps -qOnly the container IDs of running containers.
docker ps -aqContainer IDs for all containers, running and stopped.
ℹ️
Verify in Docker Desktop Too
Docker Desktop → Containers shows the same information visually: container name, image, status (running/exited), and port mappings. Clicking a container reveals its Logs, Inspect details (platform, command, state, image, and any bind mounts), Files browser, and Stats.
🔑 Connecting Into a Container — docker exec ▼
There are two ways to use docker exec: open an interactive shell and explore, or run a single one-off command directly.
1 Open an interactive shell
If the container has a shell (Linux containers usually do), connect into it:
bash
docker exec -it myapp1 /bin/sh
Once inside, you can browse and inspect the container like any Linux filesystem:
inside the container
cd /usr/share/nginx/html
ls
cat index.html
hostname
printenv
df -h
2 Run a one-off command directly
No need to enter the shell first — pass the command straight to exec:
bash
docker exec -it myapp1 ls
docker exec -it myapp1 hostname
ℹ️
The Container's Hostname = Its Container ID
Running hostname inside the container prints the container ID (e.g. ending in 04bb) — Docker sets that as the default hostname.
💡
Try It — Access the App, Then Modify It
Browse to http://localhost:8080 to see the nginx page served from inside the container. Then, purely for learning, create a new file inside the container and load it in the browser:
inside the container
echo "hello from inside" > /usr/share/nginx/html/abc.html
Then visit http://localhost:8080/abc.html — the new file is served immediately, since you edited it live inside the running container.
⚠️
Reminder — This Change Is Ephemeral
As covered in the previous lecture on Docker containers, this abc.html file lives only in the container's writable layer. Remove the container without persistent storage attached, and it's gone — that's the topic of the upcoming volumes and bind mounts demos.
⏯️ Stop, Start & Remove — stop, start, rm, rmi ▼
1 Stop a running container
bash
docker stop myapp1
# docker ps -a now shows it as "Exited (0) ... seconds ago"
2 Start it again
bash
docker start myapp1
3 Remove a stopped container
bash
docker stop myapp1
docker rm myapp1
After this, both docker ps and docker ps -a show nothing — the container is fully gone.
4 Force-remove a running container in one command
rm -f stops and removes a container in a single step:
bash
docker rm -f myapp1
5 Remove the image
bash
docker images
docker rmi stacksimplify/my-nginx:v1
ℹ️
Name or ID — Your Choice
Every command above (stop, start, rm, exec) accepts either the container name (e.g. myapp1) or its container ID — whichever is more convenient at the time.
💡
Docker Desktop Has Buttons For This Too
Docker Desktop lets you stop, start, and restart containers with a click — but the important skill to build is the command-line versions, since that's what you'll rely on in real environments (servers, CI pipelines, remote hosts) where there's no UI.
