🔎 Finding the Image on Docker Hub ▼
The demo image is stacksimplify/my-nginx — a public image on Docker Hub, so no login or registry credentials are required to pull it.
1 Search on hub.docker.com
Go to hub.docker.com and search stacksimplify/my-nginx. Click the repository to see its overview page.
2 Click the "Tags" tab
The Tags tab lists every published version of the image: v1, v2, v3, and v4.
3 Check platform compatibility
This image is built for both amd64 and arm64 architectures — a multi-platform image. Docker automatically pulls the right variant for your machine's CPU architecture. Building multi-platform images is covered later, in the advanced sections of the course.
Diagram 1 — Each tag (v1–v4) is a separate build; each build supports both amd64 and arm64 architectures.
⚠️ Docker Hub Pull Limits & the Alternative Registry ▼
⚠️
Docker Hub Enforces Image Pull Limits
Docker Hub limits how many times you can pull images within a time window. Limits vary by account plan, and once a repository (shared across many students and courses) hits its daily limit, further pulls fail until the limit resets.
Docker Hub PlanPull Limit
Personal (free)200 image pulls per 6 hours
Pro5,000 image pulls per day
ℹ️
Alternative — GitHub Container Registry (no pull limits)
The same my-nginx image is also published to GitHub Packages under the stacksimplify org, with the same v1–v4 tags and no download limits.
Docker Hub
🏷️ stacksimplify/my-nginx:v1
⚠️ Subject to per-plan pull limits
🔓 Public — no login required
GitHub Container Registry
🏷️ ghcr.io/stacksimplify/my-nginx:v1
✅ No download/pull limits
🔓 Public — no login required
💡
Use This If Docker Hub Pulls Fail
If you hit a "toomanyrequests" / rate-limit error pulling from Docker Hub, switch to the GHCR mirror by changing the image reference from stacksimplify/my-nginx:v1 to ghcr.io/stacksimplify/my-nginx:v1.
⬇️ Pulling the Image ▼
1 Run docker pull
Copy the pull command straight from the Docker Hub tag page, or type it manually:
bash
docker pull stacksimplify/my-nginx:v1
2 Wait for the download
Docker downloads each image layer, then prints a confirmation:
output
v1: Pulling from stacksimplify/my-nginx
Status: Downloaded newer image for stacksimplify/my-nginx:v1
3 Verify with docker images
List local images to confirm the pull succeeded:
bash
docker images
You should see stacksimplify/my-nginx listed with the v1 tag, its image ID, and its size.
4 Confirm in Docker Desktop
Open Docker Desktop → Images → Local tab to see the same image listed with full details. If you're logged into Docker Hub inside Docker Desktop, the Hub tab shows every image in your Docker Hub account (all repos you have access to), while the Local tab shows only what's actually downloaded to your machine.
