📄 What Do COPY and ADD Actually Do? ▼
Both instructions share the same core definition: copy new files or directories from a source and add them to the image's filesystem at a destination path.
1 COPY
Copies files or directories from the build context, a build stage, a named context, or even another image — into the image being built.
2 ADD
Does everything COPY does (copying from the build context), plus two extra abilities: it can fetch files from a remote URL, and pull from a Git repository. ADD can also automatically extract local tar archives into the destination.
ℹ️
In Practice
In roughly 99.99% of real-world Dockerfiles, COPY is used. ADD is reserved for the rare cases where you specifically need tar auto-extraction or a remote fetch.
⚖️ COPY vs ADD — Full Comparison ▼
AspectCOPYADD
File transfer from build context✅ Yes✅ Yes
Tar archive extraction❌ Not supported✅ Automatically extracts local .tar files
Remote URL support❌ Not supported✅ Can fetch a file from a URL
PerformanceFaster, lower overheadSlower, especially with tar/URL features
SecuritySafe — local file transfers onlyRiskier — pulling from remote URLs introduces risk
Recommended use casePreferred for all local file copiesOnly when tar extraction or URL fetching is absolutely needed
⚠️
The Rule of Thumb
Use COPY unless you specifically need tar extraction or fetching from a remote URL — then, and only then, reach for ADD.
🧱 The Dockerfile — COPY + ADD Together ▼
Two things get added to the base image: a single file via COPY, and an entire tarred folder of files via ADD (which auto-extracts them).
Dockerfile
FROM nginx:alpine-slim

# OCI labels (trimmed down for this demo)
LABEL org.opencontainers.image.authors="Your Name"
LABEL org.opencontainers.image.title="Demo5 Dockerfile ADD vs COPY"
LABEL org.opencontainers.image.description="ADD vs COPY instruction demo"
LABEL org.opencontainers.image.version="1.0"

# COPY: a single local file
COPY copy-file.html /usr/share/nginx/html/

# ADD: a local tar archive, auto-extracted into the destination
ADD static-files.tar.gz /usr/share/nginx/html/
1 app-files/ folder
Contains index.html plus file1.html through file5.html — index.html links out to each of the five files.
2 copy-file.html
A single standalone file, copied directly with COPY — no compression involved.
3 static-files.tar.gz
All six files from app-files/ compressed into one archive, then extracted automatically into the image by ADD.
🔨 Build, Run & Verify ▼
1 Create the tar archive
From inside app-files/, compress every file into a single archive:
bash
tar -czvf static-files.tar.gz index.html file1.html file2.html file3.html file4.html file5.html
Then move (or copy) static-files.tar.gz into the same folder as the Dockerfile, next to copy-file.html.
2 Build the image
Run this from inside the folder containing the Dockerfile, copy-file.html, and static-files.tar.gz:
bash
docker build -t demo5-dockerfile-add-vs-copy:v1 .
3 Run it
bash
docker run --name my-add-vs-copy-demo -p 8080:80 -d demo5-dockerfile-add-vs-copy:v1
4 Verify in the browser
Visit http://localhost:8080 — the index page (delivered via ADD's extraction) links to file1.html through file5.html. If the page looks stale, hard-refresh — browsers cache aggressively. Then visit http://localhost:8080/copy-file.html directly to confirm the COPY-delivered file is present too.
5 Verify inside the container
bash
docker exec -it my-add-vs-copy-demo /bin/sh
cd /usr/share/nginx/html
ls
You should see copy-file.html (from COPY) alongside index.html, file1.html…file5.html (extracted by ADD) — all sitting side by side in the same directory.
Diagram 1 — COPY moves one file as-is; ADD extracts the tar archive's contents into the same destination.
🧹 Cleanup & Summary ▼
1 Remove the container and image
bash
docker rm -f my-add-vs-copy-demo
docker rmi demo5-dockerfile-add-vs-copy:v1
✅
Summary

COPY and ADD both move files from a source into the image's filesystem — but ADD can additionally fetch from a remote URL or a Git repo, and auto-extract local tar archives.

COPY is faster, safer, and simpler — it's the right default in nearly every case. Reach for ADD only when you specifically need tar extraction or a remote fetch.
