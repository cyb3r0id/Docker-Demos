📄 What Is a Dockerfile? ▼
Definition: A Dockerfile is a simple text file that contains a set of instructions for building a Docker image. It automates the process of creating images by specifying each step explicitly.
1 Create a "Docker Files" folder
Keep your Dockerfile-based projects organized — e.g. a folder named docker-files for this demo.
2 Name the file exactly Dockerfile
Capital D, no file extension: Dockerfile. This is the filename Docker looks for by default when you run docker build.
3 Define instructions inside it
Each line — like FROM or COPY — is called an instruction. Instructions run in order, top to bottom, each one building on the previous step.
ℹ️
Why Use a Dockerfile?
Anything you'd want an image to do — install software, copy files, set environment variables, run setup commands, or launch your application — has to be defined somewhere. That "somewhere" is the Dockerfile. It's why Dockerfiles are often called the heart of Docker.

🧱 This Demo's Dockerfile ▼
FROM nginx — take the public nginx image as the base. On top of it, COPY a custom index.html into the location nginx serves files from.
Dockerfile
FROM nginx
COPY index.html /usr/share/nginx/html/
1 FROM nginx
Every instruction after this line builds on top of the official nginx image — you inherit its OS layer, nginx binary, and default config.
2 COPY index.html /usr/share/nginx/html/
Copies your own index.html (sitting next to the Dockerfile) into nginx's default web root, overwriting the default placeholder page.
ℹ️
Confirm the Base Image on Docker Hub
Search nginx on hub.docker.com — that's the official image this Dockerfile is based on.
🔄 Build → Test → Tag → Push — the Full Workflow ▼
Diagram 1 — Every custom image goes through this same build-test-tag-push cycle before it's shareable from Docker Hub.
1 Build the image
Run this from inside the docker-files folder, where the Dockerfile lives:
bash
docker build -t my-nginx-custom:v1 .
-t names and tags the resulting image. The trailing . means "use the current directory as the build context" — that's where Docker looks for the Dockerfile and any files referenced by COPY.
2 Run & test it
bash
docker run --name custom-test -p 8080:80 -d my-nginx-custom:v1
Visit http://localhost:8080 and confirm your custom index.html is being served instead of the default nginx welcome page.
3 Tag it with your Docker Hub ID
Before pushing, the image needs to be tagged with your Docker Hub username (the video uses stacksimplify as an example — substitute your own ID):
bash
docker tag my-nginx-custom:v1 <your-dockerhub-id>/my-nginx-custom:v1
4 Log in and push
bash
docker login
docker push <your-dockerhub-id>/my-nginx-custom:v1
5 Verify on Docker Hub
Log in at hub.docker.com and confirm the new repository and tag appear under your account.
6 Bonus: docker search
A quick way to look up images from the CLI without opening a browser:
bash
docker search nginx

Summary

A Dockerfile is a plain-text set of build instructions, named exactly Dockerfile.

A minimal example: FROM a base image, then COPY in your own files.

docker build creates the image → docker run tests it → docker tag prepares it with your Docker Hub ID → docker push publishes it.
